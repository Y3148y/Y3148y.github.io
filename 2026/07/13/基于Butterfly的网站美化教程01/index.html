基于Butterfly的网站美化教程01

发表于2026-07-13|更新于2026-07-13|技术学习
|浏览量:

前言

在上一篇文章 基于 GithubPage 和 hexo 的简单个人网站搭建教程 我们已经初步搭建好了简单的博客网站，接下来我们开始试着丰富和美化这个网站。本文将介绍使用Gitcus增加网站评论区，增加菜单栏，修改图片布局。

评论

技术选型

主要有Waline（轻量级，评论无需登录，免费额度）,Gitalk（基于 GitHub Issue + Preact ）,Twikoo（腾讯云，有免费额度）,Giscus（GitHub 账号评论，评论存在仓库 Discussion）。
本站采用了Giscus，因为前两种都没有成功(╥﹏╥)，Gitalk 存在网络卡顿问题，Vercel 站点访问超时。

Waline(Valine官方升级版)

是 Valine 作者迭代的新版本，兼容 Valine 的使用习惯，支持匿名留言（昵称 + 邮箱即可），不用 GitHub 账号；国内 CDN 加速，访问稳定性远超 Gitalk
免费部署在 Vercel，自带评论后台、邮件提醒、反垃圾评论，侧边栏最新评论卡片可适配。

登录 Vercel并创建项目

打开 https://vercel.com/ ，使用你的 GitHub 账号登录（和博客同一个 GitHub 账号即可）
登录完成后打开官方一键部署链接：
https://vercel.com/new/clone?repository-url=https%3A%2F%2Fgithub.com%2Fwalinejs%2Fwaline%2Ftree%2Fmain%2Fexample

创建完成之后会出现一个Go to Dashboard 点金进入项目控制台

绑定Neon数据库

左侧导航栏点击 Storage → Create Database，如果没有就点击搜索栏搜索Storage。
数据库服务商选择 Neon → Continue

弹出 Neon 授权窗口，点击 Accept and Create，全部选项保持默认直接下一步
数据库创建完成后，回到 Vercel 项目页面，点击顶部 Deployments → Redeploy 重新部署一次，让数据库配置生效
部署完成后，点击页面右上角 Visit，复制这个地址https://xxx.vercel.app

Butterfly 主题完整配置 _config.butterfly.yml

Gitalk

Gitalk 是基于 GitHub Issue + Preact 开发的免费开源评论插件。

  • 注意：
    首次打开文章必须管理员手动登录初始化 Issue，否则评论框空白。

新建专门存放评论的公开仓库

仓库必须设置Public=，否则无法读取评论。
settings->开启Issues功能(默认开启)

创建 GitHub OAuth App（获取 ClientID / ClientSecret）

GitHub 头像下拉 → Settings → 左侧底部 Developer settings → OAuth Apps → New OAuth App；
表单严格填写：
Application name：Gitalk-Blog（自定义即可）
Homepage URL：你的博客地址 https://你的用户名.github.io
Authorization callback URL：和上面完全一样 https://你的用户名.github.io
Application description：可空
点击 Register application 创建；
复制页面上方 Client ID；
点击 Generate a new client secret，生成密钥并立刻复制保存（刷新页面会消失，只能生成一次）。

Twikoo

Twikoo 依托腾讯云上海节点，国内访问零超时、无 Vercel 休眠卡顿，免费额度足够个人博客使用。

开通腾讯云云开发环境

  1. 打开腾讯云官网 https://cloud.tencent.com/，完成账号登录（微信 / QQ 扫码即可，新用户有免费资源）
  2. 顶部搜索产品「云开发 CloudBase」进入控制台，点新建环境
  3. 环境配置填写：
    • 环境名称：自定义（如blog-comment-twikoo）
    • 计费方式：选按量付费（个人博客极低流量不会扣费，有永久免费额度）
    • 所属区域：固定选上海 ap-shanghai（国内延迟最低）
  4. 点击确定创建，等待 1 分钟环境初始化完成，复制 2 个关键参数留存：
    • 环境ID(envId)：一串形如xxx-123456的字符
    • 区域：ap-shanghai

一键部署 Twikoo 函数

  1. 在刚建好的环境侧边栏 →「静态网站」，开启静态网站服务（默认开关关闭，手动点开）
  2. 切换到「函数」标签页，点「新建」→「一键部署三方模板」，在模板列表选中 Twikoo，确认部署
  3. 等待部署完成，无需手动修改函数配置，Twikoo 后端服务就绪

配置安全域名

  1. 进入云开发控制台：「安全配置」→「Web 安全域名」
  2. 填入两类域名：
    • 线上博客域名：Y3148y.github.io（你的 GitHub Pages 地址）
    • 本地调试域名：localhost（本地 hexo 预览提交评论用）
  3. 保存规则，拦截非法站点调用接口

Giscus

完全免费、依托 GitHub 仓库存储评论（Discussions 板块），在 GitHub 仓库安装 Giscus App 完成仓库授权。

创建仓库

随便创建一个仓库用来放评论（可以复用你博客XXX.github.io仓库，也可以新建一个专门存放评论的仓库），然后给仓库开启 Discussions 功能：打开目标仓库 → 顶部Settings → 下滑找到「Features」，勾选 Discussions 启用讨论板块

安装 Giscus GitHub 应用（仓库授权）

  1. 打开授权页面：https://github.com/apps/giscus
  2. 点击右侧Install按钮，选择仓库授权范围：
    • 选「Only select repositories」，勾选你开启 Discussions 的评论仓库，完成授权
  3. 权限仅读取、写入 Discussion 数据，不会拿到仓库代码修改权限

生成 Giscus 配置参数（填到yaml文件的）

打开 Giscus 配置主页：https://giscus.app/ ，逐项填写表单：

  1. Repository：下拉选中授权好的 GitHub 仓库（格式：用户名 / 仓库名，示例Y3148y/Y3148y.github.io）
  2. Discussion Category（讨论分类）：下拉推荐选Announcements（评论归类到公告分区，方便后台统一管理）
  3. 页面下方「Discussion Mapping（文章和讨论匹配规则）」，选最通用的：Page pathname ↔ Discussion title（按文章网页路径自动匹配评论）
  4. 按需开启拓展选项：
    • 勾选「Enable reactions for the main post」：给评论开启点赞表情互动
    • Language 选zh-CN适配中文界面
  5. 表单填完，找到代码 JavaScript 代码，有4 个关键参数（记住待会要考）：
    data-repo：仓库标识
    data-repo-id
    data-category
    data-category-id

提醒

Giscus中，Unsuspend your installation，代表Giscus GitHub App 授权处于被暂停状态，GitHub 拦截了所有接口调用，会无法评论。

修复：

  1. 点击右侧黑色按钮 Unsuspend，完成二次确认，恢复 Giscus 对仓库的读写权限
  2. 点上方绿色Save保存仓库授权配置
