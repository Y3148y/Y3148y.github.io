基于GithubPage和hexo的简单个人网站搭建教程

发表于2026-07-10|更新于2026-07-10
|浏览量:

前言

个人博客效果展示：听雨

为什么要搭建个人博客平台？

  • 无平台限制、永久属于自己：CSDN、掘金、公众号等平台，文章会被限流、删帖、收录受限，甚至账号封禁内容全无。个人博客是完全私有化的知识库，自己全权掌控。

  • 零成本永久上线：无需购买服务器、无需域名费用、无需续费，GitHub Pages 终身免费托管，学生、新手零负担。

  • 极简高颜值、可自由定制：告别平台统一模板，自定义壁纸、动效、看板娘、导航、页脚，打造专属个人IP主页。

  • 助力学习与求职：记录编程、学习笔记、生活随笔，积累技术沉淀，求职时可作为个人作品集，大幅加分。

本文为零基础保姆级全套教程，无需代码基础、无需服务器，全程 Windows 通用。包含：环境安装→博客初始化→主题配置→全套炫酷美化→看板娘部署→页面搭建→博客写作规范→上线部署个人博客。

一、技术栈选型

  • 选择 Hexo 静态博客框架，基于 Node.js 运行，全程配置化操作，无需手写代码，上手简单。
  • 运行环境 Node.js + Git：Hexo 无法直接在电脑运行，没有环境就无法生成网站、无法安装插件、无法部署上线。支撑 Hexo 所有命令运行、插件安装、网站部署，是搭建博客的必备前置工具。
  • Butterfly主题模板，自带暗黑模式、动态背景、烟花/粒子动效、访客统计、看板娘适配、友链页面等上百种功能，无需自研，开箱即用。
  • 托管平台：GitHub Pages 免费静态资源托管服务，把做好的网站传上去，所有人就能在线访问你的博客。

二、前期准备

  • 注册 GitHub （代码托管平台）账号并验证邮箱（国外平台若网络受限打不开可使用steam++）

  • 安装 Node.js 或 Git (版本控制工具，Windows用户需要)等基础工具

  • 准备 Markdown 编辑器（如 VS Code、Typora）

2.1 注册 GitHub 账号（代码托管平台）

GitHub 是全球最大的代码托管平台，我们用来免费存放、上线个人博客，是搭建博客的核心账号。

注册要求：

  1. 必须验证邮箱，否则无法使用部署功能

  2. 网络卡顿无法打开官网，可使用网络加速工具优化访问

后续操作：注册完成后，新建公开仓库，仓库名严格填写：你的用户名.github.io
注意开启GitHub Page，顶部Settings->左侧导航栏Pages：在 Branch 部分，确保分支选为 main（或 master），目录选 / (root)。点击 Save。

2.2 必备工具安装（Node.js + Git）

这两个工具是搭建博客的地基，缺一不可，下面附官方下载地址、安装细节、验证命令、名词详解。

2.2.1 Node.js 安装

Node\.js 是 Hexo 博客的「运行底座」，没有它，电脑无法识别 Hexo 命令，无法生成网站、无法安装任何美化插件。

选型理由：优先安装 LTS 长期稳定版，兼容性最强、无版本报错、适配所有 Hexo 版本，新手百分百适配。

官方下载地址：https://nodejs.org/zh-cn/

安装核心注意事项：

  • 下载后双击安装包，全程默认下一步，不要修改安装路径、不要取消任何组件

  • 安装程序会自动配置电脑环境变量，无需手动操作，零基础无脑装

Git 是「命令操作+代码部署工具」，我们所有的安装插件、更新博客、上线网站操作，都需要通过它执行命令完成。

选型理由：Windows 系统原生没有命令部署功能，必须依靠 Git 实现 Hexo 全套操作，是博客搭建、更新、上线的必备工具。

官方下载地址：https://git-scm.com/download/win

十一、新手高频报错&一站式解决方案

  • 看板娘不显示：缓存未更新，执行 hexo clean 重新生成部署即可

  • 页面404：未创建对应页面文件夹，或页面头部缺少 type 配置

  • 修改配置不生效：必须执行 clean 清理缓存，单纯生成文件不更新配置

  • 手机端无特效：主题配置中对应动效的 mobile 参数改为 true
